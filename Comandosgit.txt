# Atividade prática 01 - Versionamento básico local

### Atividade Prática: Versionamento local de um Projeto Simples com Git

**Objetivo:** Criar e versionar localmente um projeto simples em JavaScript utilizando Git.

**Passos:**

1. **Configuração Inicial do Git**
    - Abra o terminal e configure seu usuário no Git:
        
        ```bash
        git config --global user.name "Seu Nome"
        git config --global user.email "seuemail@exemplo.com"
        ```
        
2. **Criar o Projeto**
    - Crie uma pasta para o seu projeto:
        
        ```bash
        mkdir projeto-simples
        cd projeto-simples
        ```
        
    - Dentro dessa pasta, crie um arquivo `index.js` e adicione o seguinte código JavaScript simples:
        
        ```jsx
        console.log("Hello, Git!");
        ```
        
3. **Inicializar um Repositório Git**
    - Inicialize um repositório Git na pasta do projeto:
        
        ```bash
        git init
        ```
        
4. **Verificar o Status do Repositório**
    - Confira o status do repositório para ver quais arquivos foram criados e ainda não estão sendo versionados:
        
        ```bash
        git status
        ```
        
5. **Adicionar Arquivos ao Versionamento**
    - Adicione o arquivo `index.js` ao controle de versão:
        
        ```bash
        git add index.js
        ```
        
6. **Criar um Commit**
    - Faça o primeiro commit do projeto:
        
        ```bash
        git commit -m "Primeiro commit - adiciona o arquivo index.js"
        ```
        
7. **Adicione uma dependência**
    - Instale a dependência do **prompt-sync** no projeto
        
        ```bash
        npm i prompt-sync
        ```
        
    
8. **Adicionar um `.gitignore`**
    - Crie um arquivo `.gitignore` na pasta do projeto e adicione o nome de algum arquivo ou pasta que você deseja ignorar no versionamento (exemplo: `node_modules/`).
    - Confira o status para garantir que o `.gitignore` foi criado corretamente:
        
        ```bash
        git status
        ```
        
9. **Commit do `.gitignore`**
    - Adicione o `.gitignore` ao versionamento e crie um novo commit:
        
        ```bash
        git add .gitignore
        git commit -m "Adiciona o arquivo .gitignore"
        ```
        
10. **Commit a dependência do prompt-sync**
    - Adicione o arquivo de dependencias e o consolide:
    
    ```bash
    git add .
    git commit -m "Adiciona prompt-sync"
    ```
    
11. **Consultar Ajuda**
    - Utilize o comando `git help` para explorar outras funcionalidades do Git e aprender mais comandos que podem ser úteis no futuro.

git checkout codigo > volta a versão anterior.

exemplo: 
71f1ff8 (HEAD -> master) Add nova atualiza
15f1c27 Add gitignore
30a4f84 Add package.json files
c7d5dce Add hello message

git checkout 30a4f84 > volta para o penultimo commit

git checkout main > volta no tempo para futuro e restaura todos commits, em alguns projetos e master

git checkout . > com ponto ele limpa tudo e volta para o que vc ja vercionou.


git tag v0.2.0 > adiciona uma tag ao commit

git tag -a v0.1.0 -m "Add tag v0.1.0" 30a4f84 > consegue direcionar uma tag em um tempo especifico do commit

git show v0.1.0 > consegue verificar o logo de quem fez algum commit

git tag -d v0.1.0 > remov tag de versão especifica 


git show id (o numero inicial do projeto no local do id ex 30a4f84) > ele especifica no log do prompt o que foi alterado

git branch -m master ou main > troca nome da principal

git branch dev > cria uma branch com nome dev
git checkout dev > você entra na branch dev e cria dentro dela com base nela


git checkout -b feature/fibonacci > você cria a branch e ja entra nela 

para fazer comando a seguir, precisa entrar na branch que vai receber (caso tenha algo q n subiu, ele nao vai deixar deletar, so se confirmar com D maiusculo -D)
git merge feature/fibonacci > a master por exemplo, vai copiar tudo que atualizou dentro da feature/fibonacci

git branch -d nome > deleta a branch

git stash > salvo algo para voce seguir depois, caso n pode comitar no momento, mas precisa alterar algo e subir rapido. (mas quando usar git checkout dev, para voltar ele some a alt)

pelo fato de sumir a alt e voltar o anterior, você precisa usar esse comando
git rebase main > ele vai pegar a alteração que vc fez para subir rapido, e atualiza na branch atual de trabalho, para seguir o projeto

git stash list > lista as areas guardadas para futuro
git stash apply 0 > pega todas
git stash drop 0> apaga
